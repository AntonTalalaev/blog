JavaScript Event listeners

A few simple words about event listeners on example of “Coloured Blocks” Assignment.

Posted by Anton Talalaev on November 29, 2020

In today’s post, I would like to give a short overview of JavaScript event listeners and one of the projects that I made during the course JavaScript Basics by Manitoba Institute of Trades and Technology (MITT). The project was called “Coloured Blocks”.

Description of the Project

The “Coloured Blocks” project represents a web page with a table of blocks. Blocks can be dynamically colored in random colors when you click it one by one, or you can use a color picker to choose a color by yourself.

The main focus of the project was to use one of the most important JavaScript web-technologies - event listeners. So, in this article, I will focus on JavaScript and event listeners.

Event listeners

Event listeners in JavaScript are special objects that can “listen” browsers events e.i. clicking the button, mouse moving, etc. You can use event listeners to add any dynamic functionality to the page. Event listeners “listen” to the special events and when the event is invoked (or “emitted”), the piece of code related to this event will be run.

Basically, any event listener works as a simple if-statement.

If-statement structure:

if (condition)

// code to run when the condition is true

}

Event listener structure:

element.addEventListener ('click' function (event) {

// code to run when particular element of the page is clicked

});

Possible solution

The “Coloured Blocks” project, as any other coding task, can be solved in many ways. The main part of the project was changing the color of blocks. In order to do so, you can create event listeners for each block. In the current task, there were only 20 blocks on the page. It’s not much. But what if you will have 100 blocks? Or 1000? In this case, you already can’t afford to create an event listener for each block. You will need to find another solution.

Event bubbling

I will not tell you much about the event bubbling, but I will try to deliver the main thought of such a principle in simple words. Each element on the page can contain other elements. In the case of the “Coloured Blocks” project, you can find that blocks elements are contained in the table element.

On this diagram, you can see that the ‘click’ event was made under the block. But the block itself is located inside the table. It means that any event that happened with blocks happened inside the table and can be ‘heard’ by the table event listener.

Now, instead of creating event listeners for each block, you can create one event listener only for the table element. All events that happen with elements inside the table element also happen with the table element.

Let's look at code example of such event listener:

tableElement.addEventListener ('click' function (event) {

if (event.target.tagName === 'DIV') {

bgChangeFunction(event.target)

}

});

  • tableElement - table element.
  • addEventListener() - function that adds event listeners to the particular element (in this case to the table element).
  • ‘click’ - a type of event that a new event listener will “listen”.
  • function (event) { … } - peace of code that will run when an event will happen.
  • if (event.target.tagName === 'DIV') { … } - here I check if actually clicked one of the blocks (all blocks are ‘DIV’ elements).
  • bgChange(event.target) - will call the function that will change the background color of the element.

As a conclusion

“Coloured Blocks” project was a very good project to understand how to use JavaScript event listeners in a more efficient way. Event listeners are an extremely important technology for the modern Internet. Using event listeners you can add to your web resource any dynamic functionality that you want. Modern frameworks use event listeners to create dropdowns, dynamic menus, chat popups, games, etc.
