What is SSL and TLS
What differences between HTTP and HTTPS

Posted by Anton Talalaev on November 22, 2020

Nowadays, online fraud such as credit card fraud, account takeover, promo abuse, etc. began more and more popular. Here I will cover web security-related topic that definitely will be useful to know about.

Probably you wondered, why do sites with http:// in the address (URL) cause an unsafe connection alert in the browser, as opposed to sites with https:// in the address?

In simple terms, the absence of ‘s’ in http:// means that the site you visit cannot guarantee the safety of your data transmitted to the site. The https:// in the address of the site indicates that data transmission between the user (client) and the site (server) is carried out using a secure encryption protocol.

In order to understand this issue, I propose to dig a bit dipper.

Hypertext Transfer Protocol (HTTP) - the basic protocol for transferring data between a user (client) and a web-site (server). HTTP describes things like request and response, sessions, cache, authentication, and more.

Using HTTP, data is sent between network users in plain text, allowing users to see the content. However, having intercepted such data, the attacker does not have any difficulties in interpreting it. This raises a security issue. So, in 1994 Netscape Communications developed and introduced HTTP Secure (HTTPS), which allows a client and server to first establish an encrypted connection and then send HTTP messages over it. HTTPS is not a separate protocol. This is normal HTTP, that works over SSL (Secure Sockets Layer) and TLS (Transport Layer Security) encrypted transport mechanisms.

Differences between SSL and TLS

SSL is a cryptographic protocol for secure communication. Since version 3.0 SSL has been replaced by TLS, but the name of the previous version stuck, therefore today SSL is most often referred to as TLS. Today, TLS and SSL technologies are used to guarantee encryption of traffic between client and a server.

How does it all work?

The principle of SSL and TLS is the same. On top of the data transfer protocol, an encrypted channel is established, within which data is transferred using the HTTP protocol. Only the machine that established the connections can decrypt the transmitted data. For everyone else who receives the transmitted packets (e.g. intruders), this information will be meaningless if they cannot decrypt it.

All sites using SSL certificates ensure that the site supports three important features at once:

  • Confidentiality
  • Integrity
  • Identification

SSL certificates are issued by a publicly trusted certificate authority (CA). CA are industry-trusted organizations that issue Internet certificates. They are responsible for the quality of service and provide guarantees of the reliability of the SSL certificate. If the user suffers due to the vulnerability of the certificate, he will be paid monetary compensation.

Each browser contains a list of trusted CAs. Sites with SSL certificates from this list are considered safe for the user.

(section Settings - Privacy and security - security - manage certificates - system roots)

If the site's SSL certificate was issued by a valid certification center, the user's browser recognizes it as trusted (authentication) and encrypts its data using the public key. Even if an attacker intercepts the information transmitted by the user (for example, credit card details for paying for an order), he will not be able to read it, since he does not have a private key to decrypt the data.

Let's summarize

Safe and efficient transmission of data over open communication channels is a critical component of a modern IT system, and SSL provides this security.
